Full topology

The complete request path

A browser loads the React SPA over managed-TLS HTTPS; the SPA calls the FastAPI backend with a Bearer JWT, gated by local Auth + RBAC. The backend's topology-assembly engine merges discovery, overlay, diff and live enrichment — drawing its shape from a runtime Azure Files manifest. Two paths then cross into Google Cloud: the GCP platform APIs are read keylessly through the Workload Identity Federation spine, while the ColdVault product API is reached over HTTPS with an API key for health, metrics and streaming AI analysis.

Pillars

Four ideas hold the system together

01 No stored keys

Key-less cross-cloud trust

The container runs as an Azure Managed Identity. A broker exchanges a short-lived Entra token at Google STS to impersonate a read-only service account — no service-account keys live anywhere.

02

Least privilege by construction

Federated credentials authorize read-only access — Logging, Monitoring, Asset Inventory, Compute, GCS and Secret Manager. The estate is observed, never mutated.

03

Generic image, runtime estate

The container image stays sanitized and generic. The real topology is injected at runtime from an Azure Files manifest share — so one image can describe any estate.
